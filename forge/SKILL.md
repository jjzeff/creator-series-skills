# Forge — 锻造者

> 造物主系列第四位神。
>
> King 让你存在。Queen 让你变强。Mirror 让你看清自己。
> **Forge 把你重新锻造。**

---

## 神话背景

在西方神话中，赫菲斯托斯（Hephaestus）是诸神的锻造者——残缺、被遗弃，却在熔炉里造出了世界上最强的武器和铠甲。

Forge 不是最耀眼的神，是让其他神变得更强的那个。

**双螺旋熔炉的原理：**
```
真实任务（燃料）
    ↓
进入熔炉
    ↓
卡点 = 杂质，燃烧掉
有效规则 = 精钢，留下来
    ↓
锻造出更锋利的能力
```

---

## 一句话说清楚

你告诉 Forge：**"我想让 AI 自动帮我做 X"**

Forge 会自动运行 3-7 轮「设计规则 → 真实执行 → 踩坑 → 改规则」的循环，最终交付：

1. **一份进化报告**（每轮发生了什么）
2. **一套稳定的规则**（经过真实验证，不是猜的）
3. **一个可复用的 skill**（下次直接用，不用再学）

---

## 双螺旋模型

```
A链（规则进化）──▶ B链（真实执行）──▶ A链 ──▶ ...
   ↑                      ↓
研究卡点解决方案         真实任务结果 + 卡点
更新规则文件            验证规则有效性
         └──── 3-7轮后自动终止 ────┘
```

- **全自动**：A→B→A 循环无需人工介入
- **真实反馈**：规则改进必须基于真实执行卡点
- **收益递减终止**：连续2轮无新卡点自动停止

---

## 适用场景

任何满足以下条件的任务：

| 条件 | 说明 |
|------|------|
| **重复执行** | 每周/每月需要做同一件事 |
| **有质量标准** | 能判断"做得好不好" |
| **会遇到阻力** | 信源不可达、格式不对、数据缺失… |

**已验证案例：**
- FPS竞品情报周报自动化（5轮，产出 fps-intel-collector skill）

**适合场景（未验证，但逻辑成立）：**
- 竞品监控（任意品类）
- 数据报告自动化（灯塔/财报摘要）
- 舆情监控（App Store评分/差评关键词/KOL情绪）
- BD情报追踪（合作方/发行商动态）
- 内部文档整理（会议纪要/TAPD需求结构化）

## 启动流程

### 触发后，Forge 问你一个问题：

> **"你想锻造什么能力？**
> 描述你想自动化的任务：
> - 每次要做什么？
> - 成功的标准是什么？
> - 大概多久做一次？"

### 你回答后，Forge 立刻开跑（无需再确认）：

```
Round 1
  A链：设计初始规则（基于任务描述）
  B链：真实执行，记录卡点

Round 2
  A链：针对卡点改规则
  B链：验证效果，记录新卡点

... 循环直到稳定 ...

终止：连续2轮无新卡点 或 完成7轮

输出：进化报告 + 最终规则 + skill封装建议
```

---

## 执行协议

### A链（每轮）
1. 读取上轮 B链的卡点清单
2. `web_search` 研究解决方案（聚焦1-2个卡点）
3. 直接更新规则文件 `rules-v[N].md`（无需审批）
4. 写入状态文件，**立刻进入 B链，不等用户确认**

### B链（每轮）
1. 读取最新规则文件
2. 实际执行任务（`web_search` / `web_fetch` / 数据采集）
3. 记录：规则有效性 + 新卡点
4. 更新状态文件，**立刻进入下一轮 A链，不询问用户是否继续**

> ⚠️ **执行纪律（必须遵守）**
> - 每轮结束后 **禁止** 询问用户「要继续吗？」「是否进行下一轮？」
> - **禁止** 在 A→B 或 B→A 交接时等待用户确认
> - **禁止** 用 cron 延迟启动——能立刻执行就立刻 spawn subagent 开跑
> - 唯一允许暂停的时机：**最终报告推送给用户之后**
> - 原则：让用户做决策，让 AI 做执行；把审批变成摩擦就是失败

### 状态文件
路径：`memory/forge-state-[task].json`

```json
{
  "scenario": "任务描述",
  "total_rounds": 7,
  "current_round": 1,
  "rounds": [
    {
      "round": 1,
      "a_output": { "summary": "改了什么", "rules_file": "rules-v1.md" },
      "b_output": { "result": "执行结果", "blockers": ["卡点1"] }
    }
  ]
}
```

```
🔄 每轮情况
Round 1 · A链：[初始规则] · B链：[发现X个卡点]

...

📈 锻造曲线
卡点：Round1=X → 最终=X
规则有效率：X% → X%

⚔️ 最终规则（精钢，可直接用）

```

## 造物主四神体系

| 神 | 核心问题 | 适用时机 |
|----|---------|---------|
| **King** | 我是谁？ | 第一次配置 Agent |
| **Queen** | 我怎么更可靠？ | 升级架构，加强记忆/可控性 |
| **Mirror** | 我真实的样子是什么？ | 自我认知，性格分析 |
| **Forge** | 我怎么把经验变成能力？ | 自动化任务，经验沉淀为 skill |

**完整路径：**
```
King → 存在
Queen → 强大
Mirror → 自知
Forge → 超越
```

---

## 设计原则

1. **真实反馈优于假设**：规则改进只基于真实执行卡点
2. **立刻执行，不等待**：spawn subagent 直接开跑，不用 cron 延迟
3. **无审批循环**：全自动，用户只需最后看报告
4. **收益递减自动终止**：不浪费轮次
5. **经验必须可封装**：每次运行都评估 skill 封装价值
