# STATE_M：MemPalace 记忆层

> 解决的问题：**AI 怎么不忘事、有历史依据**
> 来源：MemPalace 框架 × Harness × 小风团队实战
> 产出：Wing 结构设计 + Palace 初始化 + Cron 改造

---

## 开场白

> "第二层：**MemPalace 记忆层**
>
> 升级公式：
> `Agent = Model + Harness + Palace`
>
> 没有 Palace，你的 Agent 每次执行都是全新个体——
> 无法说「这比上周好 15%」，无法调出历史案例，口碑评级只靠当次感知。
>
> 我们来建你的记忆宫殿——"

---

## 诊断

> "快速问三个问题：
>
> 1. 你的 Agent 现在有没有「跨会话记忆」？（有 memory/ 目录，但每次启动是否知道上次做了什么？）
> 2. 你有没有历史案例库，可以用关键词检索？（比如「翻出上次版本上线时的舆情处理方案」）
> 3. 你的 Cron 任务在执行前，会不会加载历史基准数据？
>
> 回答（全有/部分有/都没有）："

## 升级 1：设计 Wing 结构

**根据 AGENTS.md 的业务角色，推导 Wing 结构：**

询问用户：
> "你的 Agent 主要处理哪类内容？
>
> A. 内容监控/报告（日报、周报、舆情）
> B. 项目管理/复盘（任务、里程碑、复盘）
> C. 研究情报（竞品、市场、数据分析）
> D. 其他（请描述）"

根据回答，生成推荐 Wing 结构（以 A 为例）：

```
palace/
├── wing_daily        ← 日报/周报存档
├── wing_crisis       ← 舆情案例库（核心价值）
├── wing_competitor   ← 竞品监控基准
└── wing_baseline     ← 口碑/健康指数基准线
```

其他场景（B/C/D）参考：
- B（项目管理）：wing_projects / wing_decisions / wing_retro
- C（研究情报）：wing_research / wing_competitor / wing_insights
- D（自定义）：引导用户描述「我最常需要检索什么历史信息」→ 每类一个 Wing

**规则提醒：**
> "Wing 数量建议 ≤ 10 个，精简优于全覆盖。
> 每个 Wing 对应一个人或一个项目维度。"

---

## 升级 2：四层记忆加载策略

**在 MEMORY.md 开头追加说明：**

```markdown
## 📐 Palace 四层记忆架构

| 层级 | 内容 | 大小 | 触发 |
|------|------|------|------|
| L0 | 身份：Agent 是谁，监控/负责什么 | ~50 token | 每次必加 |
| L1 | 关键事实：活跃项目、当前风险、重要决策 | ~120 token | 每次必加 |
| L2 | 场景记忆：近期会话、当前项目细节 | 按需 | 话题出现时 |
| L3 | 深度检索：语义查全库 | 按需 | 明确需要时 |

每次启动只加载 L0+L1（~170 token），按需深入。
```

**询问用户：**
> "你的 MEMORY.md 目前是否按这个结构组织了？
> 帮你自动重构成四层结构吗？（是/否）"

如果是：读取 MEMORY.md，重构为 L0/L1/L2 三段结构，L3 指向 `memory_search`。

## 升级 3：Cron 改造——加入记忆加载和写入

**询问：**
> "你有几个 Cron 定时任务？
> 说出关键任务的名称，我来帮你改造 Prompt，加入记忆加载和写入。"

**为每个 Cron Prompt 追加两段（示意）：**

```
【记忆加载 — 执行前必做】
1. 读取 MEMORY.md（L0+L1）— 确认当前重点和活跃风险
2. memory_search "[本次主题] 历史记录" — 调出历史对比数据，评级有历史依据
3. memory_search "竞品 近期动态" — 只记录变化量，不重复描述静态数据

【记忆写入 — 执行后必做】
1. 将本次执行摘要写入 memory/YYYY-MM-DD.md
2. 如遇 L1+ 事件，写入 memory/palace/wing_crisis/YYYY-MM-DD.md
3. 更新 MEMORY.md 中的「活跃风险快照」
```

告知用户：
> "✅ 记忆加载和写入已加入 Cron Prompt 模板。
> **关键设计**：记忆加载在 Prompt 开头，记忆写入在 Prompt 结尾，AI 从「执行者」变成「学习者」。"

## 升级 4：基准线自建机制（不依赖外部接口）
