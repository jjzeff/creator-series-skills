## 诊断

逐一询问：

### Q1（经验沉淀）

> "你的 Agent 执行完一个任务后，**有没有把「这次遇到了什么坑」记录下来**？
>
> 还是做完就结束，下次遇到同样的边界判断，还是重新想？
>
> （有记录/没有）"

### Q2（案例库）

> "你有没有一个**可以用关键词检索的案例库**？
>
> 比如：遇到舆情 L2 事件，能马上调出上次同类事件怎么处理的？
>
> （有/没有）"

### Q3（Skill 迭代）

> "你的 Skill 文件（SKILL.md 或 AGENTS.md 里的角色规则），
> **有没有版本号？有没有记录「上次更新了什么」？**
>
> （有版本管理/没有）"

## 升级 1：Cron 加「学习尾巴」

**为每个核心 Cron Prompt 追加学习回路段落：**

```
【学习回路 — 每次执行后必做】
本次执行是否遇到以下任意一种情况：
  ① 边界判断（产品边界、数据来源、内容分类等，哪怕判断正确也要记）
  ② 数据异常（±10% 突变、平台无数据、接口失败等）
  ③ 评级边界模糊（L0/L1 之间难以判断）
  ④ 新的处理方式（比以往更好的做法）
  ⑤ 不确定判断（感觉不对但不知哪里不对）

→ 有：写入 memory/learnings/YYYY-MM-DD.md（格式见下）
→ 没有：跳过，不写文件

【learnings 记录格式】
日期：YYYY-MM-DD HH:MM
任务：[Cron 名称]
类型：[边界判断/数据异常/评级边界/新方式/不确定]
情况：[一句话描述遇到了什么]
处理：[怎么做的]
结论：[下次遇到同样情况应该怎么做]
```

**关键设计说明：**
> "「哪怕判断正确也要记」——正确判断里的推理是经验，不只是结果。
> 路径独立：`memory/learnings/` 与日常日志 `memory/YYYY-MM-DD.md` 分开，不混。"

创建 `memory/learnings/` 目录（如不存在）。

## 升级 2：案例库结构化

**询问：**
> "你的 Agent 有没有「高价值事件」需要沉淀到案例库？
>
> 比如：处理过的危机、重要决策案例、版本复盘要点。
>
> 有的话，描述一下是什么类型的案例？"

**根据回答，创建案例库目录和模板：**

```
memory/palace/wing_{{CASE_TYPE}}/
```

每个案例用以下模板写入：

```markdown
# 案例：{{标题}}

**日期**：{{DATE}}
**等级**：L0 / L1 / L2 / L1+
**触发点**：{{什么情况触发了这个案例}}

## 处置过程
{{处置步骤，简洁记录}}

## 结论
{{这个案例说明了什么，下次遇到类似情况直接这样做}}

## 关键词
{{搜索这个案例时应该用什么关键词}}
```

**从现有日志中挖取历史案例（可选）：**
> "要我从你的 `memory/` 目录里自动挖取历史 L1+ 事件，转成案例格式吗？（是/否）"

如果是：读取最近 30 天的 `memory/YYYY-MM-DD.md`，找出所有包含「L1+」「L2」「错误」「教训」「血泪」关键词的条目，转化为案例格式写入案例库。

## 升级 3：Skill 版本治理

**创建 `memory/skill-registry.md`：**

```markdown
# Skill 版本台账

> 维护规则：每次 Skill 更新后，这里必须同步更新版本号和变更说明。
> AutoDream 每两周检查一次，高频问题触发 Skill 升版。

| Skill 名称 | 当前版本 | 上次验证 | 状态 | 备注 |
|-----------|---------|---------|------|------|
{{填入用户现有的 Skill 列表}}
```

**扫描工作区，列出所有已安装的 Skill：**

读取 `~/.openclaw/skills/` 目录，列出所有 Skill 名称，填入台账。

**询问：**
> "你的哪些 Skill 最重要（每天都在用）？
> 我来给它们加版本号，并写入台账。"

对用户指定的核心 Skill，在其 SKILL.md frontmatter 中追加版本注释：
```yaml
# version: 1.0.0
# last-updated: YYYY-MM-DD
# changelog: 初始版本 / [最近改动一句话]
```

## 升级 4：配置 AutoDream Cron（学习聚合）

> "AutoDream 是 Learning Loop 的核心引擎——定期聚合 learnings，触发 Skill 升版本。"

**询问：**
> "你想多久让 AutoDream 跑一次？
>
> A. 每两周（推荐，给 learnings 积累足够数据）
> B. 每周
> C. 每月
>
> （A/B/C）"

**创建 AutoDream Cron，Prompt 核心内容：**

```
AutoDream — Skill 进化聚合

【STEP 1】读取 memory/learnings/ 下最近 {{周期}} 天的所有文件

【STEP 2】统计高频问题
对所有条目按「类型」分类统计：
- 同类问题出现 ≥ 3 次 → 标记为「需要 Skill 改进」
- 输出：高频问题列表（类型 + 出现次数 + 示例）

【STEP 3】Skill 改进建议
对每个「需要改进」的 Skill：
- 找到对应 SKILL.md 或 AGENTS.md 中的角色规则
- 追加「注意事项」段落（不删原有内容）
- 版本号 +0.0.1，写入 memory/skill-registry.md

【STEP 4】Skill 健康检查（每 ≥ 12 天触发一次）
- 检查 skill-registry.md 中标记为「高频使用」的 Skill
- 核查上次验证日期，超过 {{周期}} 天未验证则标记「需要复核」

【STEP 5】输出 AutoDream 报告
写入 memory/YYYY-MM-DD.md（当天日志），格式：

--- AutoDream 报告 ---
检查周期：{{START}} ~ {{END}}
learnings 条目：{{总数}}
高频问题（≥3次）：{{列表}}
触发 Skill 升版：{{列表，无则写「无」}}
需要复核的 Skill：{{列表，无则写「无」}}
---------
```
