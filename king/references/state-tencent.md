## 如果用户跳过

---

## 如果用户确认（是/腾讯/是的）

### Step T1：安装 iWiki MCP

用 `knot_skills` 搜索并安装 iWiki skill：

```
action: search
query: iwiki 文档 企业知识库
```

找到后执行安装，告知用户：
> "✅ iWiki MCP 安装完成，{{AGENT_NAME}} 现在可以直接读写你的 iWiki 文档了。"

---

### Step T2：询问 iWiki 目录位置

> "你想把 {{AGENT_NAME}} 的知识库建在 iWiki 哪里？
>
> - 说「**我的个人空间**」→ 建在你的 iWiki 个人空间根目录下
> - 粘贴一个 **iWiki 页面链接**（iwiki.woa.com/p/XXXXXXX）→ 建在那个目录下
> - 说「**跳过**」→ 以后自己手动建"

---

### Step T3：构建 iWiki 目录结构

读取已生成的 `AGENTS.md`，提取业务团队角色列表（`## 业务团队` 下所有角色名）。

**完整目录结构：**

```
主目录：{{AGENT_NAME}}-{{USER_ROLE_SHORT}}智库
  │
  ├── TEAM-{{业务角色1}}
  ├── TEAM-{{业务角色2}}
  ├── TEAM-{{业务角色3}}
  ├── TEAM-{{业务角色4}}
  │
  └── TEAM-团队管理（固定，必建）
        ├── 个人记忆名片
        └── 团队协作机制
```

**示例（小灵 · CF 运营）：**

```
小灵-CF运营智库
  ├── TEAM-策划参谋
  ├── TEAM-运营分析师
  ├── TEAM-市场情报官
  ├── TEAM-内容创作者
  └── TEAM-团队管理
        ├── 个人记忆名片
        └── 团队协作机制
```

---

### Step T4：创建目录内容

**执行顺序：**

1. 用 `mcporter call iwiki createDocument` 创建主目录页（作为导航首页）
2. 创建各 TEAM 子目录页
3. 创建 TEAM-团队管理 目录页
4. 在团队管理下创建「个人记忆名片」页
5. 在团队管理下创建「团队协作机制」页
