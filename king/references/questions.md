# 引导问题树

---

## STATE_A：Soul — 建立身份定位

### 目标
生成一段清晰的身份定位声明，格式为：
> 我是 [用户身份]，你是专为 [服务对象] 服务的 [定位]，所有回答围绕 [业务范围] 展开，风格 [调性]，精准响应 [核心需求]。

这是 SOUL.md 的核心，比"性格：严谨"更有实际意义。

---

### A1 - 用户身份（开场直接问，已在开场白问了）

收集用户回答，例如：
- "我是 CF 运营负责人" → USER_ROLE = CF运营负责人
- "我是游戏发行 PM" → USER_ROLE = 游戏发行项目经理
- "我是独立开发者" → USER_ROLE = 独立开发者

如果用户回答模糊（如"我做游戏的"），追问一次：
> "具体是哪个方向？运营、研发、发行、还是别的？"

---

### A2 - Agent 的服务对象和定位

> "好的。那你希望这个 Agent 专门服务于什么团队/业务？"

示例引导：
- CF 运营负责人 → "是专门服务 CF 运营团队，还是更广的腾讯游戏运营？"
- 独立开发者 → "是你个人的全能助手，还是专注某个项目？"

收集：SERVICE_SCOPE（服务对象）

---

### A3 - 核心业务范围

> "它主要帮你处理哪些事？列几个关键词就行。"

示例：
- CF运营 → "活动策划、内容宣发、赛事播报、玩家运营、数据分析"
- 发行PM → "版本规划、市场投放、竞品情报、跨部门协同"
- 开发者 → "代码管理、技术方案、Bug追踪、架构优化"

收集：BUSINESS_SCOPE

---

### A4 - 风格调性

> "它说话的风格呢？比如：专业严谨、贴合游戏调性、简洁直接、温暖亲切……"

如果用户说不出来，给出默认值：
> "默认用：专业、简洁、直接，不废话。你觉得合适吗？"

收集：TONE_STYLE

---

### A5 - Agent 名字（最后问）

> "最后，给它起个名字吧？"

（放在最后，因为用户这时候对 Agent 已经有感觉了，更容易起出有意义的名字）

收集：AGENT_NAME

---

### 生成预览（STATE_A 结束前展示）

> "好，我来给它生成身份定位，你看看对不对："

```
【{{AGENT_NAME}}】的身份

我是{{USER_ROLE}}，你是专为{{SERVICE_SCOPE}}服务的智库 Agent，
所有回答围绕{{BUSINESS_SCOPE}}展开，
风格{{TONE_STYLE}}，精准响应{{USER_ROLE}}的相关需求。
```

> "这样可以吗？还是需要调整？"

确认后 → 写入 SOUL.md → 进入 STATE_B

---

## STATE_B：Agents — 构建核心团队

### 开场白
> "SOUL.md 写好了 ✅ {{AGENT_NAME}} 知道自己是谁了。
> 下一步，给它配一支团队。
> 先告诉我，你日常工作中最需要哪方面的支持？"

---

### B1 - 中台三角（默认必选，无需询问，直接告知）

> "每个团队都有三个默认角色，这是基础，不可缺少："

| 角色 | 职责 |
|------|------|
| **CTO** | 技术升级、工具选型、系统维护 |
| **HR** | 团队管理、定期复盘、绩效回顾 |
| **PMO** | 流程监管、任务跟踪、Gate 审核 |

"这三个我已经帮你配好了。"

---

### B2 - 业务角色推荐

根据 STATE_A 收集的 USER_ROLE + BUSINESS_SCOPE，推荐 1-3 个定制角色：

**推荐映射（内部逻辑，不展示给用户）：**

| 用户场景 | 推荐角色 |
|---------|---------|
| 游戏运营/发行 | 情报官（竞品监控）、活动策划官（节奏把控） |
| 内容创作/制片 | 制片人（内容节奏）、编剧（创意发散） |
| 代码/工程开发 | 架构师（技术方案）、测试官（质量把关） |
| 商务/BD | 客户经理（关系维护）、策略官（方案设计） |
| 通用个人助理 | 研究员（信息整理）、日程官（时间管理） |

> "根据你的工作，我推荐再加：[角色列表]
> 合适吗？还是想换别的？或者直接跳过，之后随时可以加。"

---

### 生成预览 → 确认 → 写入 AGENTS.md → 进入 STATE_C

## STATE_C：Memory — 建立记忆系统

### 开场白
> "团队配好了 ✅ 现在给 {{AGENT_NAME}} 建记忆。
> 记忆让它每次对话都知道你是谁、在做什么。"

**C1 - 活跃项目**
> "你现在最主要在跟进哪些项目或任务？每个一句话就行。"

**C2 - 需要记住的偏好/背景**
> "有没有什么背景信息是它必须知道的？比如：你的团队规模、工作节奏、特别注意事项……"

**C3 - 记忆结构**
> "不同项目的资料需要分开存吗，还是放一起就好？"
- 分开 → 告知可后续扩展 memory/palace 结构
- 放一起 → 扁平结构，MEMORY.md + 日期流水

### 生成预览 → 确认 → 写入 MEMORY.md → 进入 STATE_D

---

## STATE_D：Heartbeat — 定时情报站

### 开场白
> "记忆建好了 ✅ 最后一步，也是很重要的一步：
> **让 {{AGENT_NAME}} 主动帮你盯信息。**
> 大多数人用龙虾做信息检索——热搜、竞品、数据……这些不应该等你来问，应该定时自动跑。"

**D1 - 信息检索类**
> "你希望它定时帮你盯什么？"
- 行业热搜 / 舆情
- 竞品动态（App 更新、发布、公告）
- 特定关键词新闻
- 数据报表 / 排行榜
- 其他（用户自填）

**D2 - 任务提醒类**
> "有没有需要定时提醒你做的事？比如每周项目复盘、每日信息整理……"

**D3 - 频率**
> "多久跑一次？"

### Cron 说明
> "好，我帮你把这些配成 Cron 定时任务，后台自动跑，不用手动触发。"

生成对应 Cron 配置写入 HEARTBEAT.md。

### 生成预览 → 确认 → 写入 HEARTBEAT.md → 进入 STATE_P2
